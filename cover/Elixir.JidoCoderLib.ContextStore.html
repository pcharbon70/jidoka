2 @moduledoc """
3 GenServer that owns and manages ETS tables for high-performance shared state.
4
5 This GenServer creates and owns three ETS tables:
6 1. `:file_content` - Cached file contents
7 2. `:file_metadata` - File metadata tracking
8 3. `:analysis_cache` - Analysis result cache
9
10 ETS tables provide O(1) access time and support concurrent reads without locks.
11
12 ## Session Scoping (Phase 3.5)
13
14 All cache operations support session-scoped data using composite keys.
15 Each session can have its own isolated cache for the same file.
16
17 * **Optional session_id:** If not provided, uses `:global` for shared cache
18 * **Composite keys:** `{session_id, file_path}` for file operations
19 * **Session isolation:** Data from one session doesn't affect another
20
21 ## Table Definitions
22
23 ### :file_content
24 Caches file contents to avoid repeated disk reads.
25
26 * **Type**: set
27 * **Key**: `{session_id, file_path}` tuple
28 * **Value**: `{content, mtime, size}` tuple
29 * **Concurrency**: read_concurrency: true
30
31 ### :file_metadata
32 Stores metadata about files (language, line count, etc.).
33
34 * **Type**: set
35 * **Key**: `{session_id, file_path}` tuple
36 * **Value**: metadata map
37 * **Concurrency**: read_concurrency: true
38
39 ### :analysis_cache
40 Caches analysis results to avoid redundant computation.
41
42 * **Type**: set
43 * **Key**: `{session_id, file_path, analysis_type}` tuple
44 * **Value**: `{result, timestamp}` tuple
45 * **Concurrency**: read_concurrency: true, write_concurrency: true
46
47 ## Examples
48
49 Cache a file (global scope):
50
51 JidoCoderLib.ContextStore.cache_file("/path/to/file.ex", "content", %{language: :elixir})
52
53 Cache a file for a specific session:
