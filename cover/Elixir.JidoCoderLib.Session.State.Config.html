3 Struct and functions for managing session state.
4
5 This module defines the data structures that hold session-specific
6 state and configuration, including validation, state transitions,
7 and serialization.
8
9 ## Session Status
10
11 Sessions can be in one of the following states:
12
13 | Status | Description |
14 |--------|-------------|
15 | `:initializing` | Session is being created, SessionSupervisor starting |
16 | `:active` | Session is ready and processing |
17 | `:idle` | Session is active but no current activity |
18 | `:terminating` | Session is being shut down |
19 | `:terminated` | Session has been shut down |
20
21 ## State Transitions
22
23 Valid state transitions:
24
25 ```
26 :initializing → :active
27 :initializing → :terminated (startup failure)
28 :active → :idle
29 :idle → :active
30 :active → :terminating
31 :idle → :terminating
32 :terminating → :terminated
33 ```
34
35 ## Examples
36
37 Creating a new session state:
38
39 {:ok, state} = Session.State.new("session-123", %{
40 llm_config: %{model: "gpt-4"},
41 metadata: %{project: "my-project"}
42 })
43
44 state.session_id
45 #=> "session-123"
46
47 state.status
48 #=> :initializing
49
50 Transitioning state:
51
52 {:ok, updated_state} = Session.State.transition(state, :active)
53
54 Serializing state:
55
56 map = Session.State.serialize(state)
57 {:ok, restored} = Session.State.deserialize(map)
